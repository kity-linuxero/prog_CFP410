Modularización

Modularización

  • Los problemas del mundo real suelen ser muy complejos.
  • Es necesario dividir el problema el sub-problemas mas simples. "Divide y vencerás"
  • Técnica: Top-Down analizar el problema, dividirlo en sub-problemas y reiterar el método en cada subprograma

Ventajas de la Modularización

  • Mayor legibilidad: Nos podemos concentrar en solo una parte del problema y es mas fácil de entender y que otros entiendan el código.
  • Mayor productividad: Es posible que se le asignen los subproblemas a diferentes personas, con lo que se podría llegar a la solución final de una forma más rápida
  • Calidad al programa: Si tenemos bien modularizado nuestro programa da calidad al software y también es parte de las buenas prácticas de programación.
  • Mantenimiento: Básicamente es corregir errores y modificar el código. Un programa modularizado será mas fácil de mantener
  • Reusabilidad: Cuando se escribe un programa a menudo es deseable reutilizar el código ya escrito y probado.

Modulos

Llamaremos módulos a cada "pedazo" de programa que resuelve una parte del problema. Como si fuera un sub-programa del programa entero.

Para hacerlo aún mas interesante, dotaremos a los módulos con instrucciones y datos propios (locales).

De esta manera tendremos módulos (sub programas) y el programa principal o (main).

Módulos

Generalidades

  • Cabecera: Se define Define el nombre del módulo. A veces se utilizan palabras clave para especificar que es un módulo. (def)
  • Tipo de valor de retorno: Se define el tipo de datos que va a devolver. Depende el lenguaje. (Opcional)
  • Lista de parámetros: Son "variables" por la cual se transfiere información al módulo. Pueden ser de entrada y/o salida. (Opcional)
  • Datos locales: Variables, declaraciones, tipos de datos, etc. Que son para uso local. (Opcional)
  • Bloque de código: Es el código que se va a procesar dentro del módulo. Puede ser tan amplio como queramos.

Ejemplo de módulo en Python


							# Modulo o sub programa llamado saludo
							def modulo_prueba(): # Cabecera
								print ("Llamaste al modulo de prueba") # Código
							# ...
							# Programa principal
							print ("Acá estamos en el programa")
							modulo_prueba() # Se invoca al módulo

Resultado


							Acá estamos en el programa principal
							Llamaste al modulo de prueba

¿Cómo modularizar en un programa?

  • Funciones/procedimientos: Parte de código que resuelve una parte del problema. Opcionalmente puede tener datos de entradas y/o salida.(Parámetros, valor de retorno)
  • Archivos externos: Es posible tener partes del código fuera de nuestro programa principal. Los archivos externos pueden importarse fácilmente.
  • Clases, objetos: Con POO es posible dividir soluciones de un problema con las diferentes técnicas que ofrece el paradigma orientado a objetos.
  • Librerías externas: Existen diversas librerías de aceso libre que pueden resolver una parte de nuestro problema.

Parámetros

Parámetros

Son variables que transfieren información entre los módulos o el programa principal. Los parámetros son opcionales.

Los módulos o funciones pueden tener muchos parámetros y un valor de retorno o salida (opcional)

Lo veremos con ejemplos.

Parámetros (cont)

Los parametros pueden ser:

  • Por valor: Se envía una copia del dato al módulo. Si el dato cambia en el módulo, el dato en el módulo que llamó no se verá afectado.
  • Referencia: Se envía "el mismo dato" (una referencia). De manera que si hay un cambio en el módulo cambiará también el dato en el otro módulo.
  • Por defecto: Algunos lenguajes permiten parámetros por defecto. En caso que no se envíen parámetros, puede definirse un valor por defecto.

Parámetros en Python

En Python no se concibe la dialéctica "paso por valor/referencia" porque el lenguaje no trabaja con el concepto de variables sino objetos y referencias.

Básicamente siempre se pasa por valor a no ser que sea una lista, arreglos, etc.

Parámetros (cont)

Envío de datos al módulo


							def modulo_saludo(nombre): # Cabecera con un parámetro
								print (f"¡Hola {nombre}!") # Código del módulo
							# ...
							# Programa principal
							nom1 = "Cristian"
							modulo_saludo(nom1) # Invocación del módulo
							nom2 = "Laura"
							modulo_saludo(nom2)

Resultado


								¡Hola Cristian!
								¡Hola Laura!

Alcance de los datos

Alcance de los datos

Como vimos anteriormente, los datos pueden ser constantes o variables. Pero también hay que tener en cuenta el alcance de los mismos dentro del programa.

Alcance de los datos (cont)

En esta instancia (para no complicarnos tanto), podemos diferenciar los datos como de alcance "global" o "local".

  • Global: Será accesible desde todo el programa y los módulos
  • Local: Será solo accesible desde el módulo donde se declaró

Variables locales y globales

¿Por qué no ponemos todas variables globales y listo?

  • No es una buena práctica: A menos que sea necesario tener una variable global, siempre se recomienda usar parámetros para enviar datos a los módulos.
  • Poca legibilidad: Un programa que está constantemente llamando variables globales puede ser poco legible para quien esté mirando el código. El mantenimiento puede verse afectado.
  • Mayor uso de memoria: Una variable global estará ocupando memoria durante toda la ejecución del programa, aunque ya no se use.
  • Atenta a la protección de datos: Si una variable es local a un módulo, solo un módulo la podrá ver. De manera que se protegen los datos ante el acceso o modificación no deseada. Si es global, en cambio cualquier parte del programa puede acceder a los datos.

Ejemplo modularización

Escriba un programa que compare dos números y diga cual de los dos es el mayor. Usar modularización.

Sin modularizar

				
							dinero= 900
							extraccion= int(input("Ingrese el monto que desea extraer:"))
							if extraccion <= dinero:
								dinero = dinero - extraccion
								print (f"Usted ha extraído ${extraccion}")
							else:
								print("No hay suficiente saldo para completar la operación")
							print  (f"El saldo restante es: ${dinero}")
El ejercicio corresponde al Ejercicio 10 de la práctica 1.

Ejemplo modularización

Usando modularización

				
							dinero= 900
							def haySaldoSuficienteParaExtraer(unMonto):
								return dinero >= unMonto
							def extraer(unMonto):
								global dinero
								dinero = dinero-unMonto
							
							extraccion= int(input("Ingrese el monto para extraer:"))
							if haySaldoSuficienteParaExtraer(extraccion):
								extraer(extraccion)
							else:
								print("No hay suficiente saldo para la operación")
							print  (f"El saldo restante es: ${dinero}")
